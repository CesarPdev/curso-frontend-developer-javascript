      Recomendación importantísima: No olviden ir haciendo commits para que vayan guardando todo lo que van haciendo en el repo de GitHub.

      Cada vez que finalicen algo como por ejemplo el mobile menu ya funciona bien, entonces:

      git status
      git add . (si, así con el punto para que agregue todos los archivos)
      git commit -m "[ADD] mobile menu, styles and JS to toggle submenu"
      git push origin main (Para enviar esos cambios que están en local -su PC- hacia el repositorio que tienen en GitHub).

      La idea de ir haciendo estos commits sencillos con pequeños cambios es:

      1. Que cuando necesiten hacer revisión de cambios, es más fácil ver que cada commit hace un solo cambio en una sola cosa, y no diez cambios en el footer, en el header, en unas imágenes, etc etc.
      2. Que si algo se rompe en algún momento, pueden regresar a un commit anterior y solo se deshace un pequeño cambio que hayan hecho, no diez cambios al tiempo.
